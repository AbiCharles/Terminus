We need a small stack-based bytecode virtual machine written from scratch in C. You'll build it over three milestones in `/app/vm.c`, implementing the single function `vm_run` declared in `/app/vm.h`. A non-functional stub is already in `/app/vm.c` for you to replace; do not change `/app/vm.h`. Use only the C standard library.

`/app/vm.h` is the authoritative contract: it defines every opcode, its operand encoding, the execution semantics, the limits (`VM_STACK_CAPACITY`, `VM_LOCALS_COUNT`, `VM_MAX_STEPS`), and every `VM_*` status code. Read it carefully. The VM executes a flat little-endian bytecode program over a stack of signed 64-bit integers, starting at program counter 0. `vm_run` writes each `PRINT`ed value into the caller's `out` buffer, sets `*out_len`, and returns `VM_OK` when the program executes a `HALT` or the matching `VM_ERR_*` code at the first error.

Your VM is compiled together with a separate test program (which you do not have) using `gcc` under UndefinedBehaviorSanitizer, so it must be free of undefined behavior — in particular, never read program bytes outside `[0, len)` — as well as functionally correct.

Milestone 1: handle program loading and the core stack machine. Implement the instructions `PUSH`, `POP`, `DUP`, `SWAP`, `ADD`, `SUB`, `PRINT`, and `HALT` with the exact semantics in the header (binary ops pop `b` then `a` and push `a OP b`). Decode each opcode and its immediate operand from the byte stream, advancing the program counter past the operand bytes.

Report the right status code for the error cases: an undefined opcode byte (`VM_ERR_OPCODE`); an instruction whose operand bytes run past the end of the program (`VM_ERR_OPERAND`); popping or consuming an operand that is not there (`VM_ERR_STACK_UNDERFLOW`); pushing when the stack is already at `VM_STACK_CAPACITY` (`VM_ERR_STACK_OVERFLOW`); and control reaching the end of the program without a `HALT` (`VM_ERR_NO_HALT`). You do not need the arithmetic, comparison, local, or jump opcodes yet — those come in later milestones.
