Add a `report` subcommand: `/app/quake report [filters] --out-prefix PREFIX [--targets LIST] [--window-years W]`. It accepts the same filters as the other subcommands, **first declusters the filtered events exactly as in milestone 2** (the Gardner-Knopoff windowing producing the mainshock catalog), then computes Gutenberg-Richter recurrence statistics **on that declustered mainshock catalog** and writes two deterministic files: a frequency-magnitude table at `PREFIX_fmd.csv` and a JSON summary at `PREFIX_summary.json`. Every statistic below — the bin counts, the completeness, the b-value, the catalog span and the exceedance rates — is computed over the declustered mainshocks, never the raw filtered catalog.

Bin magnitudes with a fixed width `delta_m = 0.1`; an event of magnitude `m` falls in the bin whose center is `floor(m / delta_m + 0.5) * delta_m`. Estimate the magnitude of completeness `mc` with the Wiemer & Wyss (2000) goodness-of-fit test by scanning candidate completeness values from low to high, exactly as follows:

1. The candidate completeness values are the magnitude bin centers from the lowest filled bin up to (and including) the highest bin that still leaves at least four bins — counting both filled and empty bins — at or above it. Evaluate the candidates in increasing magnitude order.
2. A candidate `Mc` is admissible only if at least 30 events lie in bins at or above it. Skip inadmissible candidates entirely.
3. For an admissible candidate with `N` events at or above it and mean magnitude `mean` over those `N` events, fit the Aki-Utsu model: `b = log10(e) / (mean - (Mc - delta_m/2))` and `a = log10(N) + b*Mc`.
4. Compute the goodness-of-fit residual `R = 100 * (1 - D / S)`, where `D` and `S` are **sums** over every bin center `M` from `Mc` up to the highest filled bin (one term per bin) with `B(M)` the observed cumulative count (events in bins whose center is ≥ `M`): `D = Σ |B(M) - 10^(a - b*M)|` and `S = Σ B(M)`. `R` is one minus the *total* absolute residual divided by the *total* observed count — an aggregate over all bins at or above `Mc`, never a single worst-bin deviation.
5. Choose `mc` as the lowest admissible candidate whose `R` is at least 90; if none reaches 90, choose the admissible candidate with the highest `R`.

Report the final b-value from the Aki-Utsu estimator applied to the mainshocks at or above `mc`, its Shi & Bolt (1982) uncertainty `sigma_b = 2.30 * b^2 * sqrt( Σ (m_i - mean)^2 / (N*(N-1)) )` over those same `N` mainshocks, and `a = log10(N) + b*mc`.

`PREFIX_fmd.csv` must have the exact header `magnitude,count,cumulative_count` and one row for every consecutive magnitude bin from the lowest filled bin to the highest filled bin of the **declustered** catalog (include every bin in that contiguous range, even empty intermediate bins, so for a lowest filled bin of 1.0 and a highest of 6.4 there are 55 rows). `count` is the number of mainshocks in that bin; `cumulative_count` is the number in that bin or any higher bin. Write the bin center with one digit after the decimal point and the two counts as integers.

`PREFIX_summary.json` must carry the keys `completeness_method` (the string `goodness_of_fit`), `gft_confidence` (90.0), `delta_m` (0.1), `n_total` (the number of declustered mainshocks after filtering), `n_above_mc` (mainshocks at or above `mc`), `mc`, `b_value`, `b_uncertainty`, `a_value`, `catalog_years` (the span from earliest to latest **mainshock** time in years, i.e. the difference of Julian days divided by 365.25, where each event's Julian day is computed from its full ISO-8601 `time` value including the time-of-day — equivalently `julianday(time)` using SQLite's `julianday()` on the complete `YYYY-MM-DDTHH:MM:SSZ` timestamp, not just the date portion), `magnitude_range` (an object with `min` and `max` over the mainshocks), `window_years` (the forecast window `W`, defaulting to 50), an `exceedance` array, and a `per_region` array. Use one digit after the decimal point for `mc`, `delta_m` and the magnitude range, and four digits for `b_value`, `b_uncertainty`, `a_value` and `catalog_years`.

The `per_region` array breaks the declustered catalog down by region. Partition the declustered mainshocks by their `region` and, for each region in **ascending region-name order**, compute the same Gutenberg-Richter statistics over that region's mainshocks. Each `per_region` object carries `region`, `n_total` (the number of mainshocks in that region), `n_above_mc`, `mc`, `b_value`, `b_uncertainty` and `a_value`, with the same numeric formatting as the top-level statistics (one decimal for `mc`, four decimals for `b_value`, `b_uncertainty` and `a_value`). A region whose mainshocks are too few to admit a valid completeness fit — the same admissibility rule as the overall statistics (at least four spanned magnitude bins and an admissible `mc` with at least 30 events at or above it) — is omitted from the array. The regions are partitioned from the single globally-declustered catalog (decluster the filtered events once, then group the surviving mainshocks by region), not by declustering each region separately.

The `exceedance` array has one object per target magnitude, in the order given. Target magnitudes come from `--targets` as a comma-separated list (for example `5.0,6.0,7.0`), defaulting to `5.0,6.0,7.0` when the flag is absent. For a target magnitude `M`, the Gutenberg-Richter fit predicts an expected event count `10^(a - b*M)` over the catalog, so the annual rate is `annual_rate = 10^(a - b*M) / catalog_years` and the Poisson probability of at least one such event within the window is `exceedance_probability = 1 - exp(-annual_rate * window_years)`. Each array object must have keys `magnitude` (one decimal), `annual_rate` (six decimals) and `exceedance_probability` (six decimals).

Both files must be byte-for-byte identical across repeated runs with the same inputs.

Exit code `2` covers every invalid invocation, with the filter-validation rules of the `query` and `decluster` subcommands plus a non-numeric `--targets` entry or a non-positive `--window-years`; validate the arguments before declustering or computing anything. Exit `3` when the declustered catalog is too small to estimate the statistics (fewer than four spanned bins, no admissible completeness candidate, or no matching events). A bad filter such as `--lat-min 200` must exit `2` (invalid), never `3`. On success exit `0`.
