Add a `report` subcommand: `/app/quake report [filters] --out-prefix PREFIX [--targets LIST] [--window-years W]`. It accepts the same filters as the other subcommands, recomputes the milestone-2 statistics on the filtered events, and writes two deterministic files: a frequency-magnitude table at `PREFIX_fmd.csv` and a JSON summary at `PREFIX_summary.json`.

`PREFIX_fmd.csv` must have the exact header `magnitude,count,cumulative_count` and one row for every consecutive magnitude bin from the lowest filled bin to the highest filled bin, using the same `delta_m = 0.1` binning as milestone 2. Include every bin in that contiguous range, even bins whose `count` is 0 — do not skip empty intermediate bins (so for a lowest filled bin of 1.0 and a highest of 6.4 there are 55 rows). `count` is the number of events in that bin; `cumulative_count` is the number of events in that bin or any higher bin. Write the bin center with one digit after the decimal point and the two counts as integers.

`PREFIX_summary.json` must carry the same statistics keys as the milestone-2 output (`completeness_method`, `gft_confidence`, `delta_m`, `n_total`, `n_above_mc`, `mc`, `b_value`, `b_uncertainty`, `a_value`, `catalog_years`, `magnitude_range`), plus `window_years` (the forecast window `W`, defaulting to 50) and an `exceedance` array. The `exceedance` array has one object per target magnitude, in the order given. Target magnitudes come from `--targets` as a comma-separated list (for example `5.0,6.0,7.0`), defaulting to `5.0,6.0,7.0` when the flag is absent. For a target magnitude `M`, the Gutenberg-Richter fit predicts an expected event count `10^(a - b*M)` over the catalog, so the annual rate is `annual_rate = 10^(a - b*M) / catalog_years` and the Poisson probability of at least one such event within the window is `exceedance_probability = 1 - exp(-annual_rate * window_years)`. Each array object must have keys `magnitude` (one decimal), `annual_rate` (six decimals) and `exceedance_probability` (six decimals).

Both files must be byte-for-byte identical across repeated runs with the same inputs. Reuse the milestone-2 numeric formatting for the shared statistics keys.

Exit code `2` covers every invalid invocation, and the filter-validation rules are exactly those of the `query` and `stats` subcommands — validate the arguments before running any statistics. That means exit `2` for an unknown flag, a missing flag value, a non-numeric value where a number is required, a min greater than its corresponding max (latitude, longitude, depth, magnitude, or start after end), a latitude outside [-90, 90] or longitude outside [-180, 180], and additionally for a non-numeric `--targets` entry or a non-positive `--window-years`. Exit `3` when the filtered catalog is too small to estimate the statistics (including no matching events), and `0` on success. A bad filter such as `--lat-min 200` must exit `2` (invalid), never `3`.
