Milestone 3: build the `component_chains` table defined in `/app/spec.md`. Contract each strongly connected component (from the `components` table you built in milestone 2) to a single node identified by its `component_id`, forming the condensation: a directed edge `A -> B` whenever some edge runs from a node in component `A` to a node in a different component `B`. The condensation is acyclic. For every component, compute `longest_chain` — the number of components on the longest directed path in the condensation that ends at that component (a component with no incoming condensation edge has `longest_chain = 1`). Write one row per distinct `component_id` back into `/app/graph.duckdb`.

The `duckdb` command-line shell is installed and on your `PATH` — run your SQL with it, e.g. `duckdb /app/graph.duckdb < your.sql`.
