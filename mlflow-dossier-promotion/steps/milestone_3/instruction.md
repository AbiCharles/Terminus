Milestone 3 — promote the compliant champion.

Apply the dossier's gates to the candidate runs you logged in milestone 2. Disqualify any candidate that misses an acceptance threshold (compare each candidate's logged `accuracy`/`f1_macro`/`roc_auc` against the policy's `metric_thresholds`), then disqualify any remaining candidate that breaches a fairness ceiling for any protected attribute. The fairness ceiling for an attribute is the `max_disparity` value inside that attribute's entry of the policy's `bias_slices` list (there is no top-level fairness-ceilings key); a candidate breaches it when its `bias_dpd_<attribute>` exceeds that attribute's `max_disparity`. From whatever survives, choose the champion as the one with the best value of the policy's ranking metric.

Register that champion's logged model — the scikit-learn model you logged for it in milestone 2 — as a new version of the policy's registered model name by calling `mlflow.register_model("runs:/<run_id>/credit_default_model", <registered_model_name>)` with the champion run's id. Use this `runs:/` run-relative URI for the model source — not a `file://` filesystem path and not a bare run path without the `credit_default_model` name. Then assign the policy's champion alias to the resulting version. `mlflow.sklearn.load_model("models:/<registered_model_name>@<champion_alias>")` must succeed and return the fitted champion estimator, and the registered version's source must reference the `credit_default_model` logged model; a registration that cannot be loaded back or whose source does not reference `credit_default_model` does not count. The disqualified candidates stay as runs but must not be registered as the champion.
