We vendored networkx into this project (it's installed editable from `/app/networkx-src`, so edits to the source take effect on import) and a regression slipped in. When we compute the longest path through a weighted DAG, the `default_weight` argument is being ignored: edges that don't carry an explicit weight attribute are always counted as weight 1, no matter what `default_weight` we pass. So the longest-path result comes out wrong whenever `default_weight` isn't 1 and some edges are unweighted. Find the cause in the networkx source and fix it so the default weight is applied. Don't change any behaviour that is already correct.
